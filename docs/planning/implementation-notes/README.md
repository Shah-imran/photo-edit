# Implementation notes (per slice)

Add one markdown file **before** coding each small vertical slice. Requirements are in [INCREMENTAL_WORKFLOW.md](../INCREMENTAL_WORKFLOW.md) **section 4** (mandatory **detailed** implementation plan, review gate, then code).

Suggested filename: `YYYY-MM-DD-short-topic.md` (e.g. `2026-05-02-library-sort-by-date.md`).

Each note must include the full checklist from section 4.1 (goal, current behavior, design, contracts, nuances, UI impact, dependencies, tests, rollback, acceptance criteria), end with **Plan approved -- implementation allowed** before coding starts, and after merge add a short **Implementation summary**.
